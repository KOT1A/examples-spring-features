Примеры использования HikariCP

## Краткое описание что такое HikariCP
HikariCP — это высокопроизводительный клиентский пул JDBC-соединений (DataSource) для Java-приложений. В Spring Boot он
чаще всего используется «из коробки» как DataSource по умолчанию. JDBC-соединение к БД дорого открывать/закрывать — пул
держит готовые соединения и раздаёт их потокам, уменьшая задержки и повышая пропускную способность.

## Как рассчитать maximumPoolSize

Простая формула:<br>
<code>maxPoolSize ≈ floor((DB_max_connections - reserved_for_other_apps) / app_instances)</code><br>
Пример: у PostgreSQL max_connections = 500, есть ещё 2 других приложения и админские подключения → reserve 50, значит на все инстансы приложения можно выделить 450. Если у тебя 10 инстансов — 450 / 10 = 45 → разумно выбрать 30–40 с учётом пиков.
Ещё ориентир: размер пула не должен быть гораздо больше числа рабочих потоков, которые реально обращаются к БД одновременно.
